Our View on Pair Programming

Pair programming is a fun and collaborative exercise. In terms of DBC, pair programming is important because it provides practical teamwork skills for when students go into the actual tech industry and job market.

Pair programming is a collaborative activity where two developers work together to code with specific roles. It is intended to maximize the efficiency and work and share knowledge.

Pair Programming Roles

  • Driver: The person who actually writes the code
  • Navigator: The person who narrates the "story" for the driver
